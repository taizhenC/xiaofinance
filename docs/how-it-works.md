# 工作原理 · How it works

一轮抓取周期（约 10–15 分钟，慢是刻意的 — 限速在保护你的账号）：

| 阶段 | 模块 | 做什么 |
|---|---|---|
| 抓取 | `infinance/providers/mediacrawler.py` | 通过 SourceProvider 接口驱动 MediaCrawler 子进程：按时间排序搜索、JSONL 输出、每轮独立日志、超时杀进程树。所有 vendor 相关机制（补丁、Cookie 传递、日志判读）都封在这个适配器里 |
| 入库 | `infinance/ingest.py` | 新鲜度闸门 #1：只有 ≤24h 的笔记/评论进入本地 SQLite |
| 去重 | `infinance/dedup.py` | 笔记 simhash 聚类 + 评论精确去重；转发簇在打分中只算一次，进入 LLM 时标注 `[×N相似]` |
| 提及识别 | `infinance/mentions.py` | 词典三档匹配：安全别名直接命中（特斯拉/巨硬/皮衣黄）；歧义别名需要附近有金融语境词（苹果+股价、老黄、马斯克）；冲突代码（LI/MS/KO…）同样需要语境 |
| 打分 | `infinance/scoring.py` | `3·帖 + 1·评 + 2·Σlog₁₀(1+帖赞) + 0.5·Σlog₁₀(1+评赞)`，按去重簇计 |
| 分析 | `infinance/analyze.py` | LLM JSON 模式逐股总结：剔除无关条目、按论点而非重复次数权衡多空、产出 summary/bull/bear/引用；引用逐字校验（见下）；输入未变则跳过 |
| 行情核对 | `infinance/prices.py` | Yahoo 免费日线（唯一非小红书外部请求，可关）；驱动「舆论与股价背离」徽章与准确率记分 |
| 黑话扫描 | `infinance/slang_scan.py` | 定期从未匹配的金融帖里挖新谐音/黑话 → 人工审核面板，接受后并入本地词典 |

新鲜度闸门 #2：所有 API 查询再按 24h 窗口过滤，内容在两次抓取之间持续老化淘汰。

## 信任机制

- **逐字引用校验（AN-01）**：LLM 给出的每条「原文引用」必须是输入内容的（归一化）子串 — 改一个字、拼接两条、纯表情都会被丢弃，必要时回退为真实高赞内容。界面上的引用要么逐字真实，要么不出现。
- **准确率记分板**：当某股分析明显偏向一边（|▲−▼|≥2），当天的「舆论倾向」会与次日/7 日实际股价对照。这是对**人群**历史判断的描述性记录，不是预测，更不是建议。
- **数据年龄可见**：每张卡都标注内容与分析的新旧。

## 边界

产品原则：**我们汇总信息，用户自己决策。** 不荐股、不给买卖信号、不按「吸引力」排序 — 只按讨论热度。架构上，MediaCrawler 永远是用户自取的外部组件（非商业学习许可），管道只依赖 SourceProvider 接口。
