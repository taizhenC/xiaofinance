# 故障排查 · Troubleshooting

第一步永远是：

```
infinance doctor
```

逐项检查环境（uv、vendor 版本锁定、Chromium、数据库、端口……），每个失败项都带修复命令。界面里的设置向导 → 环境检查是同一套检查。

## 登录问题决策树（最常见）

界面「登录与会话」面板会自动诊断并给出对应按钮；下面是同一逻辑的人读版。

```
会话状态？
├─ 「登录已失效」(expired)
│   → 点「重新登录」扫码（或 `infinance login`）。QR 要在约 6 分钟内扫完。
│
├─ 「搜索被拒绝」(unauthorized：日志出现 没有权限访问)
│   ├─ ① 账号注册在 rednote.com（国际版）？
│   │   → 国际版与 xiaohongshu.com 是两套后端（API 域名、Cookie 域都不同），
│   │     扫码成功也会被大陆后端拒绝。点「切换到国际版后端」再试。
│   │     这是海外用户最常见的原因。
│   ├─ ② 切了还不行，但浏览器里搜索正常？
│   │   → 账号健康，是二维码新建的会话不被信任。把浏览器的 Cookie
│   │     粘贴到面板的输入框（需含 a1= 与 web_session=），复用已被平台
│   │     信任的会话。
│   └─ ③ Cookie 也被拒？
│       → 账号本身被风控。验证手机号，回 App 正常用几天，**停止重试**
│         （重试提高风险分）。这不是工具能修的。
│
└─ 换浏览器不是办法：搜索请求走 httpx，跟哪个浏览器扫码无关。
```

每次修改后用 `infinance smoke`（3 帖最小抓取）验证，别浪费整轮抓取。

## 其他常见问题

| 现象 | 处置 |
|---|---|
| 打开页面提示 web UI not built | 源码安装需要构建前端：`npm ci --prefix frontend && npm run build --prefix frontend`；发布包自带 |
| 「立即抓取」被拒（间隔/预算） | 这是账号保护（见[账号安全](account-safety.md)）。间隔与预算可确认后强制；登录冷却不能强制，先重新登录 |
| 抓取半途取消后数据少 | 取消保留已入库内容，衍生统计下一轮自动补齐 — 正常 |
| 登录历史出现「未知设备」 | 已默认修复（UA 与真实系统一致）。若自定义过 `BROWSER_USER_AGENT`，改回与本机一致 |
| 端口被占 | `.env` 改 `PORT`，或找到占用者：doctor 会提示 |
| 升级后启动报数据库错误 | 见[升级指南](upgrade.md) — 迁移前有自动备份 `infinance.db.v<N>.bak` |
| 爬虫报 patch/anchor 错误 | 上游改了结构：重跑 `infinance setup` 重新锁定；仍报错说明需要新版本 infinance |
