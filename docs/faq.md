# 常见问题 · FAQ

**Q：这是投资建议工具吗？**
不是，而且刻意不是。它只回答「小红书上现在谁最热、大家怎么说」— 不荐股、不给买卖信号、不按吸引力排序。每个界面都写着：仅供参考，不构成投资建议。

**Q：需要小红书 API key 吗？**
不需要，小红书没有开放 API。数据来自你自己的账号会话（扫码或 Cookie）。因此请阅读[账号安全](account-safety.md)。

**Q：不配 DeepSeek key 能用吗？**
能。没有 key 时卡片显示热门原文引用，没有 AI 总结。配上 key（`.env` 里 `DEEPSEEK_API_KEY`）才有逐股总结/多空要点。任何 OpenAI 兼容端点都行（`LLM_BASE_URL`/`LLM_MODEL`），成本量级：每轮几美分。

**Q：为什么一轮要 10–15 分钟？**
限速是账号保护的一部分（单并发 + 每请求 3 秒 + 低采集量）。把它调快等于把你的账号往风控里送。

**Q：数据会上传到哪里吗？**
不会。一切都在本机 SQLite；对外请求只有小红书（你的会话）和可关闭的 Yahoo 行情。没有遥测。原始内容 7 天自动删除。

**Q：能在手机上看吗？**
可以：`.env` 设 `HOST=0.0.0.0` **并且必须设 `AUTH_TOKEN`**（不设会拒绝启动 — 否则局域网里任何人都能用你的账号触发抓取）。然后手机浏览器访问 `http://<电脑IP>:8000`。页面只读开放，改动操作需要令牌。

**Q：热度分怎么算的？**
`3·帖 + 1·评 + 2·Σlog₁₀(1+帖赞) + 0.5·Σlog₁₀(1+评赞)`，转发簇只算一次。细节见[工作原理](how-it-works.md)。

**Q：准确率记分板是什么？**
当分析明显偏多/偏空（|▲−▼|≥2），当天的倾向会和次日/7 日股价对照。它记录的是**小红书人群**的历史命中率 — 描述性统计，不是任何人的预测。

**Q：「示例数据 DEMO」是什么？**
没有真实数据时看板展示的内置合成数据，让你先看懂界面。所有示例文本都是编造的，不含任何真实小红书内容；抓到真实数据后自动切换。

**Q：多开几个实例可以吗？**
不要。抓取互斥、定时任务都是进程内状态，`infinance run` 固定单 worker。一台机器一份实例。

**Q：为什么不内置微博/B站？**
先把小红书的采集稳定性做扎实。管道只依赖 SourceProvider 接口，第二平台是一个适配器的事，在路线图 P2。
