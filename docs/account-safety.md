# 账号安全 · Account safety

数据来自**你自己的小红书账号**。诚实的风险陈述：自动化访问违反平台条款，账号存在被限制的可能。本工具的每个默认值都在压低这个风险，但无法把它降到零。

## 工具对你的账号做什么 / 不做什么

**做：** 以你的会话身份进行关键词搜索、读取笔记与评论 — 相当于一个手速稳定、频率很低的真人浏览。

**不做：** 不发帖、不评论、不点赞、不关注、不私信；不改任何账号设置；Cookie 与会话只保存在本机（不进日志、不进命令行参数、界面永不回显）。

## 内置保护（默认开启，DC-03）

| 机制 | 默认 | 说明 |
|---|---|---|
| 单并发 + 3 秒间隔 | 固定 | 爬虫层面每请求礼貌延时 |
| 低采集量 | 每关键词 12 帖 | 一轮约 100–200 帖 |
| 最小抓取间隔 | 45 分钟 | 间隔内手动抓取需要显式确认 |
| 每日请求预算 | 15000 条/24h | 计入每轮的笔记+评论页数；超限后定时任务自动跳过 |
| 登录失效冷却 | 30 分钟 | 会话被判失效后**禁止**继续抓取（强制也不行）— 对被风控的会话反复重试只会提高风险分。重新登录成功立即解除 |
| 可见浏览器 + 真实 UA | 自动 | 登录用可见窗口；User-Agent 与你的操作系统一致，避免「未知设备」记录 |

调整入口在 `.env`（`MIN_FETCH_GAP_MIN` / `DAILY_REQUEST_BUDGET` / `AUTH_COOLDOWN_MIN`）。调高前想清楚：预算的意义就是在你误配置关键词时先于平台发现问题。

## 强烈建议

1. **用小号。** 注册一个专用账号，正常使用几天（浏览、点赞、验证手机号）再接入工具。
2. **别追求实时。** 默认 5 小时一轮已经足够描述「今天散户在聊什么」。把间隔改小并不会让信息更有价值，只会让账号更显眼。
3. **登录失败就停。** 三类失败各有明确处置（见[故障排查](troubleshooting.md)）；「多试几次」在风控体系里是负分项。
4. **子评论开关慎用。** `ENABLE_SUB_COMMENTS=true` 会成倍增加每帖请求数。

## 数据去哪了

原始内容只存本机 SQLite，7 天自动删除；运行记录与分析 30 天。对外请求只有两类：小红书（你的会话）与 Yahoo 行情（可用 `ENABLE_PRICE_QUOTES=false` 关掉）。没有任何遥测。
